שלב 8 – Gatling Max Limit

1. התקנת Gatling

  1. הורד את Gatling Open Source:
    👉 Gatling Download

  2. חלץ את הקובץ לתיקייה במחשב שלך, למשל:
    C:\gatling\gatling-charts-highcharts-bundle-<version>\

2. יצירת תרחיש (Simulation)

  1. ב־Gatling יש תיקייה בשם user-files\simulations.

  2. צור קובץ Scala חדש, למשל MaxLimitSimulation.scala.
    דוגמה בסיסית:

    scala
    import io.gatling.core.Predef._
import io.gatling.http.Predef._
import scala.concurrent.duration._

class MaxLimitSimulation extends Simulation {

  val httpProtocol = http
    .baseUrl("http://localhost:8080/YourApp") // לשים את ה-URL של האפליקציה שלך
    .inferHtmlResources()

  val scn = scenario("Max Limit Test")
    .exec(http("Load Homepage")
      .get("/"))

  setUp(
    scn.inject(
      rampUsersPerSec(1) to (100) during (60.seconds) // מעלה הדרגתית מ־1 עד 100 משתמשים ב־דקה
    ).protocols(httpProtocol)
  )
}

    ➝ המטרה: לראות באיזה נקודה מתחילות שגיאות (HTTP 5xx / Response Time גבוה). זה יהיה ה־Max Limit שלך.

3. הרצה ידנית

  1. פתח CMD/Terminal בספריית Gatling.

  2. הפקודה להריץ:

    bash
    bin\gatling.bat

    (ב־Linux/Mac: bin/gatling.sh)

  3. בחר את הסימולציה MaxLimitSimulation.

  4. בסוף ריצה:

    • ייווצר דוח HTML בתיקייה results.

    • פתח את index.html כדי לראות גרפים (Users, Response Time, Failures).

4. ניתוח תוצאות

  • Max Limit = כמות המשתמשים/בקשות לשנייה לפני שמופיעות שגיאות משמעותיות או זמני תגובה גבוהים מאוד.

  • בדוח תראה איפה מתחילות הבעיות – זהו ה־bottleneck של האפליקציה שלך.

5. הכנה ל־Jenkins

  1. התקן Jenkins Plugin ל־Gatling:

    • Manage Jenkins > Manage Plugins > Available > Gatling

  2. צור Jenkins Job (Freestyle או Pipeline) שמבצע:

    • Clone מה־GitHub שלך (כבר עשית בשלב 4).

    • Run Gatling simulation עם פקודת Maven/SBT/Gradle (או bin/gatling.bat -s MaxLimitSimulation).

    • לשמור את התוצאות (Jenkins מציג את הגרפים אוטומטית עם הפלאגין).
